### 📄 ТЗ: Система визуального архива камер (Часть 1 из 3)

#### 1. Общая Архитектура и Доступ к Файлам 🏗️

* **Архитектура:** Классическое клиент-серверное приложение (Frontend + Backend).
* **Доступ к хранилищу:** Бэкенд работает с путями к файлам как с локальными директориями 📁. Монтирование сетевых дисков (SMB) и авторизация делегируются операционной системе, на которой запущен бэкенд.
* **Синхронизация:** Сканирование файлов не происходит в фоновом режиме. Обновление базы запускается исключительно **вручную** (кнопка «Пересканировать» в интерфейсе) 🔄.

#### 2. Стек Технологий 🛠️

* **Frontend:** React ⚛️. Для отрисовки графиков (Heatmap) используются только бесплатные Open-Source библиотеки (например, *Nivo*, *Apache ECharts* или *Recharts*).
* **Backend:** Python 🐍 (рекомендуется *FastAPI* для быстрой асинхронной работы или *Flask*).
* **База данных:** SQLite 🗄️. Встроенная, легковесная, с возможностью полной или частичной очистки (например, сброс кэша конкретного дня).

#### 3. Конфигурация системы (YAML) ⚙️

Система конфигурируется через внешний `.yaml` файл. Это позволяет легко добавлять новые источники без изменения кода. 📝

```yaml
cameras:
  - id: "cam_garden_01"
    name: "Foscam Garden"
    path_snapshots: "\\192.168.1.99\Camera\Foscam\FI9805W_C4D6553DECE1\snap"
    path_videos: "\\192.168.1.99\Camera\Foscam\FI9805W_C4D6553DECE1\video"
  - id: "cam_garage_02"
    name: "Garage Cam"
    path_snapshots: "\\192.168.1.99\Camera\Garage\snap"
    path_videos: "\\192.168.1.99\Camera\Garage\video"

```

#### 4. Структура Базы Данных (SQLite) 🗂️

Для обеспечения мгновенной отрисовки тепловых карт (Heatmap) без задержек, схема БД должна быть максимально плоской и содержать строгие индексы. ⚡

* **Таблица `files**` (Основной архив метаданных):
* `id` (Primary Key)
* `camera_id` (String, связь с YAML конфигурацией)
* `file_type` (Enum: 'photo' | 'video')
* `file_path` (String, полный путь до исходного файла)
* `file_size` (Integer, размер в байтах)
* `timestamp` (Datetime, извлеченный из атрибутов ОС или имени файла)
* *Критично:* ⚡ Создать комбинированные индексы `(camera_id, timestamp)` и `(camera_id, file_type, timestamp)`.


* **Таблица `thumbnails**` (Кэш превьюшек):
* `id` (Primary Key)
* `file_id` (Foreign Key -> files.id)
* `thumb_path` (String, локальный путь к сгенерированной картинке-превью)
* *Связь:* При удалении записи из `files`, запись из `thumbnails` удаляется каскадно (ON DELETE CASCADE) 🗑️.
