### 📄 ТЗ: Система визуального архива камер (Часть 3 из 3)

#### 8. Пользовательский Интерфейс (UI/UX) и Дизайн 🎨

* 🌙 **Стилистика:** Строгая тёмная тема (dark mode), полностью вдохновлённая интерфейсом Home Assistant 🏡. Фон глубокий чёрный или тёмно-серый ⬛, элементы интерфейса минималистичны.
* 🔵 **Иконки:** Везде используются синие иконки стандарта Material Design Icons (MDI) 💠, чтобы приложение выглядело как родной дашборд умного дома 📱.
* 📊 **Сводка данных:** На абсолютно каждом экране (независимо от уровня вложенности) всегда закреплена статистика 📈: общий занимаемый объём памяти 💾, точное количество фото 📸 и количество видео 🎬.

#### 9. Навигация и Тепловая карта (Heatmap) 🗺️

* 🌡️ **Тепловая визуализация:** Вместо сложных пропорциональных блоков интерфейс строит Heatmap (тепловую карту) 🟦. Ячейки периодов окрашиваются в оттенки синего цвета: чем больше гигабайт за период, тем насыщеннее и ярче цвет 🌊. Это позволяет глазом мгновенно выхватить самые "тяжелые" архивы 👀.
* ⬇️ **Иерархия (Drill-down):** Пользователь логично проваливается вглубь архива 🛤️:
1. **Годы:** Квадраты доступных годов 📅.
2. **Месяцы:** 12 ячеек внутри выбранного года 📆.
3. **Дни:** Календарная сетка выбранного месяца 🗓️.
4. **Часы:** Линейная разбивка конкретного дня на 24 отрезка ⏳.



#### 10. Детальный просмотр (Уровень «Час») и Видео ⏰

* 📜 **Хронологическая лента:** Внутри выбранного часа единым списком загружаются все файлы без пагинации ♾️. Фотографии отображаются в виде сгенерированных превью 🖼️.
* 📽️ **Интеграция видео:** Видеофайлы встраиваются в эту же ленту по времени создания ⏱️, но отображаются просто крупной синей MDI-иконкой видео 🎬 (без ресурсоемкой генерации превью).
* ▶️ **Воспроизведение:** При клике на иконку видео оно открывается для просмотра прямо в браузере через стандартный HTML5-плеер 💻.
* 🗑️ **Интерфейс удаления:** При выделении файлов (или целого часа) для очистки всплывает модальное окно Safe Delete 🛑. В нём чётко списком показаны миниатюры удаляемых фото и автоматически найденные привязанные к ним видео 🔗, чтобы обеспечить полный визуальный контроль перед безвозвратным стиранием с SMB-диска 🧹.

#### 11. Режимы просмотра превью (View Modes) 👁️

Внутри уровня «Час» пользователь может переключаться между режимами отображения фотографий. Режим выбирается через выпадающий список в шапке, либо клавишей **M** (или Insert) — переключение идёт по кругу.

Выбранный режим сохраняется в `localStorage` и восстанавливается при следующем открытии.

##### Архитектура: реестр режимов

Каждый режим — отдельный файл-модуль в папке `frontend/src/components/viewModes/`. Все режимы регистрируются в `index.js`, который экспортирует массив `VIEW_MODES`. `HourViewer` не знает о конкретных режимах — он просто итерирует реестр: строит из него dropdown и вызывает `mode.getImageUrl()` при отрисовке карточки.

Интерфейс режима:

```js
{
  key: string,          // уникальный идентификатор, хранится в localStorage
  label: string,        // название в выпадающем списке
  getImageUrl(file, { pagePhotoIds, diffThreshold }): string
}
```

##### Существующие режимы

* 🖼️ **Normal** (`normalMode.js`) — стандартное превью. Запрашивает `/api/thumbnail/{id}`. Режим по умолчанию.
* 🔍 **Motion diff** (`motionDiffMode.js`) — сравнение с соседними кадрами на странице. Запрашивает `/api/diff_thumbnail/{id}?page_ids=...&threshold=...`. Порог чувствительности (`diff_threshold`, по умолчанию 20) настраивается в Tools → Diff Threshold и сохраняется в `localStorage`.

##### Как добавить новый режим

1. Создать файл `frontend/src/components/viewModes/myNewMode.js` по образцу существующих.
2. Добавить его в `frontend/src/components/viewModes/index.js` — один импорт и одна строка в массиве `VIEW_MODES`.
3. Больше ничего менять не нужно: dropdown и клавиша M подхватят новый режим автоматически.

Если новый режим требует дополнительных параметров (помимо `pagePhotoIds` и `diffThreshold`), их нужно пробросить из `HourViewer` через пропс `diffThreshold`/аналог и передать в `mode.getImageUrl()` вторым аргументом.
