#### 15. Дизайн и общие принципы UI/UX 🎨

* 🌙 **Тёмная тема**, вдохновлённая Home Assistant: глубокий чёрный/тёмно-серый фон,
  минималистичные элементы.
* 🔵 **Иконки** — синие Material Design Icons (MDI), чтобы приложение выглядело как
  родной дашборд умного дома.
* 📊 **Закреплённая сводка:** на каждом экране, независимо от уровня вложенности,
  всегда видна статистика — общий занимаемый объём 💾, число фото 📸 и число
  видео 🎬 (верхняя панель Header).
* 🎚️ Глобальный размер шрифта настраивается (CSS-переменная, 12–22 px).
* 📷 **Выбор камеры** — горизонтальный ряд «пилюль» (pill-кнопок).
* 🧭 **Хлебные крошки** показывают текущий путь погружения (All Years / 2024 / Nov / 16).

Состояние навигации (уровень погружения, выбранная камера, диапазон дат)
сохраняется в `localStorage` (ключ `nav_state`) и восстанавливается при
перезагрузке — пользователь возвращается туда, где был.

#### 16. Навигация и тепловая карта (Heatmap) 🗺️

* 🌡️ **Тепловая визуализация:** вместо пропорциональных блоков — heatmap. Ячейки
  периодов окрашиваются в оттенки синего: чем больше гигабайт за период, тем
  насыщеннее цвет (интенсивность считается из размера). Глазом мгновенно видно
  «тяжёлые» архивы.
* ⬇️ **Иерархия (Drill-down):** четыре уровня. Клик по ячейке — погружение на уровень
  ниже; крошки — возврат вверх.

  1. **Годы** — квадраты доступных годов.
  2. **Месяцы** — 12 ячеек внутри года.
  3. **Дни** — календарная сетка месяца.
  4. **Часы** — 24 отрезка дня.

* Под heatmap — столбчатая диаграмма (размер по периодам).
* Данные тянутся из `GET /stats?group_by=...`; интенсивность и форматирование
  байтов считаются на фронтенде.
* Кнопка **Пересканировать** (`POST /scan`) со спиннером; по завершении данные
  обновляются.

##### 16.1. Содержимое ячейки heatmap

* Цвет интенсивности (по объёму) + бейджи количества фото/видео + тултип.
* Полоска превью — N равномерно выбранных фото (`GET /previews`), число задаётся
  настройкой `previews_per_cell` (0 — отключить).
* Иконки объектов AI — до 5 уникальных (`GET /ai_objects_summary` для диапазона
  ячейки), обновляются по триггеру после пакетного анализа.
* На уровне «час» ячейка дополнительно подгружает `GET /distribution` и показывает
  **бейдж равномерности** (жёлтый/красный) — см. §19.

##### 16.2. Полная навигация с клавиатуры ⌨️

Стрелки перемещают фокус по сетке heatmap; Enter — погрузиться; Backspace/Esc —
вверх. Клавиатура heatmap неактивна, пока открыт просмотр часа. Также поддерживается
режим выделения ячеек (см. §17).

#### 17. Режим выделения ячеек и пакетные операции (CellSelBar) 🧩

Без входа в конкретный час можно выделить несколько ячеек heatmap и выполнить над
ними операцию (удаление или AI-анализ).

**Вход в режим выделения** (доступен на уровнях «час» и «день»):

| Действие | Эффект |
|---|---|
| Кнопка «Select hours» / «Select days» | Войти в режим выделения на уровне час/день |
| `Space` на ячейке в фокусе | Переключить ячейку + войти в режим |
| `Ctrl+A` | Войти в режим и выделить все непустые ячейки |
| Клик по ячейке (в режиме) | Переключить ячейку |
| `Esc` | Выйти из режима |

**Панель CellSelBar** (две строки):

* Строка 1 — управление выделением: `All` · `None` · счётчик/размер выделенного ·
  `Delete selected` (только на уровне «час») · `Cancel`.
* Строка 2 — AI-анализ: выпадающий список провайдера (OpenVINO / Gemini / Claude) +
  список моделей + слайдер порога (только OpenVINO, 10–80%, по умолч. 25) + кнопка
  `Analyze (N)`.

**Что делает Analyze по провайдерам:**

| Провайдер | Действие |
|---|---|
| OpenVINO | `POST /openvino_analyze_range` по каждой выделенной ячейке последовательно (прогресс `X/Y`) |
| Gemini | По 1 превью-фото с ячейки (`GET /previews`), всё пачкой → `POST /gemini_analyze_batch`. Нужен `gemini_api_key` |
| Claude | То же через `POST /claude_analyze_batch`. Нужен `claude_api_key` |

После завершения иконки в затронутых ячейках перерисовываются (триггер обновления —
счётчик, входящий в зависимости перезапроса `ai_objects_summary`).

#### 18. Детальный просмотр уровня «Час» ⏰

* 📜 **Хронологическая лента:** файлы часа загружаются сеткой карточек, **с
  пагинацией** (размер страницы настраивается). Фото — в виде сгенерированных
  превью; видео встроены по времени создания.
* 🖼️ **Карточка фото:** превью (в текущем режиме визуализации), полноэкранный
  лайтбокс по клику, наложение AI-иконок + тултип описания (в AI-режимах). Зум при
  наведении настраивается.
* 📽️ **Карточка видео:** по умолчанию (`video_preview_mode = none`) — иконка камеры
  + время, без картинки. При выбранном режиме превью — тянет `/video_thumbnail` и
  показывает JPEG или анимированный GIF.
* ▶️ **Воспроизведение видео:** клик открывает полноэкранный HTML5-плеер прямо в
  браузере. Управление: `Space` — play/pause, `←/→` — перемотка на ±1/5
  длительности, `Esc` — закрыть; есть скачивание, открытие внешне, откат на VLC.
* 🔢 **Пагинация** ленты с настраиваемым размером страницы.

> Расхождение с историческим замыслом: изначально предполагалась лента **без
> пагинации**. В готовой системе час пагинируется (`hour_page_size`, по умолч. 50).

##### 18.1. Панели режима

* **Переключатель режима** — выпадающий список в шапке (или клавиша `M`/Insert по
  кругу). Активный режим в `localStorage` (`hour_view_mode`).
* **Панель параметров режима** (для motion-режимов с настройкой) — слайдер порога.
* **AI-панель** (для `isAiMode`-режимов): компактная — метка + модель + кнопка Run в
  первой строке; порог (OpenVINO) / статистика / эмодзи во второй. Под кнопкой Run —
  агрегированные эмодзи уникальных объектов по всем фото текущей страницы.
* **Статистика запросов AI** (только отображение): число запросов «за последнюю
  минуту / за 24 ч», считается из меток времени в `localStorage`
  (`ai_requests_gemini`, `ai_requests_claude`); записи старше 25 ч отсекаются.

##### 18.2. Горячие клавиши в просмотре часа ⌨️

* `↑ ↓` — навигация по карточкам, `Space` — выделить, `Delete` — удалить, `Esc` —
  закрыть просмотр.
* Удержание клавиши peek — показать оригинал поверх обработанного превью.
* В режиме выделения — отдельный набор клавиш (выделить всё/ничего, удалить).

##### 18.3. Удаление из просмотра часа 🗑️

При выделении файлов (или целого часа/страницы) всплывает модалка **Safe Delete**:
списком показаны миниатюры удаляемых фото и автоматически найденные привязанные
видео (по относительным путям) — для полного визуального контроля перед
безвозвратным стиранием. Поток: предпросмотр (`/delete/preview`) → подтверждение
(`/delete/confirm`). Поддерживается удаление одного файла, всей страницы и всего
часа.

#### 19. Контроль равномерности (uniformity) 📈

Предупреждает, когда записи в часе подозрительно равномерно «размазаны» по минутам
(ложные срабатывания от ветра/дождя). По данным `GET /distribution` (60 минутных
корзин) считаются три метрики, каждая 0 = единичное концентрированное событие,
100 = запись каждую минуту:

| Ключ | Метка | Формула | 0 = … | 100 = … |
|---|---|---|---|---|
| `active` | AF | `nActive / 60 × 100` | 1 запись за минуту | в каждой минуте есть запись |
| `entropy` | SE | `H / log₂(60) × 100` | всё в одной минуте | идеально равномерно |
| `bc` | BC | `активные блоки / 12 × 100` | 1 из 12 пятиминутных блоков | все 12 активны |

Комбинированная оценка = `0.40×AF + 0.35×SE + 0.25×BC`.

Пороги предупреждения (жёлтый) и тревоги (красный) настраиваются для каждой метрики
(ключи `uniformity_*_warn` / `uniformity_*_alert`). Бейджи: в шапке графика
распределения — все три, всегда (зелёный/жёлтый/красный); в ячейке heatmap на уровне
час — только выбранная метрика (`uniformity_method`) и только при warn/alert.

**График распределения** — 60 столбцов по минутам; клик по столбцу прыгает на
соответствующую страницу ленты.

#### 20. Настройки (Tools modal) и хранение 🛠️

Все пользовательские настройки — в браузерном `localStorage` (на сервер/диск не
пишутся; привязаны к профилю браузера). Модалка настроек разбита на вкладки; каждая
вкладка читает/пишет свои ключи. Изменения вступают в силу немедленно.

##### 20.1. Вкладки и ключи

**General:**

| Ключ | По умолч. | Диапазон | Назначение |
|---|---|---|---|
| `font-base` | 15 | 12–22 px | Глобальный размер шрифта |
| `previews_per_cell` | 3 | 0–10 | Превью в ячейке heatmap (0 — выкл.) |

**Hour view:**

##### 20.2. Экспорт / импорт настроек (YAML)

* **Экспорт** — скачивает `snapshots-settings.yaml` со всеми настройками, **кроме**
  `nav_state` и `ai_requests_*` (это сессия/статистика, не конфигурация).
* **Импорт** — мягкий по дизайну: поля `api_key` **никогда не импортируются**
  (защита ключей); отсутствующие ключи пропускаются; неверный тип — пропускается;
  число вне диапазона — зажимается в допустимый; ошибка разбора YAML — ничего не
  применяется; неизвестные ключи игнорируются. Старые конфиги всегда применяются
  безопасно. После импорта всё вступает в силу без перезагрузки.

#### 21. Сводный список взаимодействий клиент↔сервер 🔌

В режиме разработки фронтенд проксирует `/api/*` на бэкенд (порт 8000). Весь доступ
к сети инкапсулирован в одном модуле API на фронтенде. Ключевые потоки:

* Старт → `GET /cameras`, восстановление `nav_state`, `GET /stats` для текущего
  уровня.
* Погружение по heatmap → `GET /stats?group_by=...` + по ячейкам `GET /previews`,
  `GET /ai_objects_summary`, на уровне час `GET /distribution`.
* Открытие часа → `GET /files` (страница) → превью по текущему режиму
  (`/thumbnail`, `/diff_thumbnail`, `/motion_thumbnail`, `/openvino_thumbnail` …),
  `GET /ai_analysis` для иконок, `/media/{id}` для оригинала/видео.
* AI → `POST /gemini_analyze_batch` | `/claude_analyze_batch` |
  `/openvino_analyze_batch` | `/openvino_analyze_range`.
* Удаление → `POST /delete/preview*` → `POST /delete/confirm` | `/delete/by_range`.
* Обслуживание → `DELETE /*thumbnails`, `DELETE /database`, `GET /storage_info`.
* Compute → `GET/PUT /compute/config`, `GET /compute/status`.
