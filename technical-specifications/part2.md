### 📄 ТЗ: Система визуального архива камер (Часть 2 из 3)

#### 5. Логика Backend и API (FastAPI / Flask) ⚙️

* 🔄 **Эндпоинт сканирования (`/scan`):** Запускается исключительно вручную пользователем 🖐️. Скрипт обходит целевые директории из YAML-конфига, читает системные атрибуты файлов (имя, размер, дата создания) и обновляет базу `SQLite` 🗄️. Постоянного фонового мониторинга директорий нет 🚫.
* 📊 **Эндпоинт статистики (`/stats`):** Отдает агрегированные данные для мгновенной отрисовки тепловой карты (Heatmap) 🗺️. Он группирует данные по годам, месяцам или дням, возвращая суммарный размер 💾, количество фотографий 📸 и видеофайлов 🎬.
* 🗂️ **Эндпоинт просмотра (`/files`):** Возвращает полный, хронологический список файлов для конкретного выбранного часа ⏰ абсолютно без пагинации.

#### 6. Система превью (Thumbnails) "On-Demand" 🖼️

* ⚡ **Ленивая генерация:** Превьюшки создаются сервером *только* в тот момент, когда они запрашиваются фронтендом для видимой зоны интерфейса 👀. Во время первоначального сканирования диска генерация картинок не происходит 🚫.
* 📏 **Алгоритм равномерной выборки:** Для экранов «Год», «Месяц» или «День» запрашивается репрезентативный набор кадров 🎞️. Бэкенд берет весь временной отрезок, делит его на равные интервалы и отдает: первый кадр, последний кадр и заданное количество промежуточных 🧩.
* 🎬 **Игнорирование видео:** Для экономии ресурсов видеофайлов превью **не генерируются** вообще ❌. Фронтенду отдается только стандартная системная иконка видео 📽️.
* 🧹 **Управление кэшем:** Сгенерированные уменьшенные картинки сохраняются локально, а путь к ним записывается в БД 🗄️. В API обязательно предусмотрены методы для ручной очистки кэша превьюшек (полностью или для конкретной камеры) 🗑️.

#### 7. Безопасное и синхронное удаление (Safe Delete) ⚠️

* 🤝 **Синхронизация Фото + Видео:** Фотографии и видео записи лежат в разных сетевых папках 📂. При удалении фотографии за определенное время ⏳, бэкенд автоматически ищет соответствующий видеофайл с допустимой дельтой времени (например, $\pm 5$ секунд в названии) 🔍.
* 🛑 **Подтверждение операции:** При попытке удаления бэкенд сначала формирует точный список связанных файлов (конкретные фото + найденные видео), которые будут затронуты 📝.
* ✅ **Физическое удаление:** Только после явного подтверждения пользователем файлы физически стираются с диска (по SMB) 💥. Привязанные записи из `SQLite` и кэш превьюшек удаляются каскадно 🧹.
